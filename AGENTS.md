# AGENTS.md

## Quick start
```powershell
flutter pub get
dart run build_runner build --delete-conflicting-outputs
dart format .
flutter test
```

## Commands
- **Build runner** (after `.dart` changes with `@riverpod` or `@freezed`): `dart run build_runner build --delete-conflicting-outputs`
- **Watch mode**: `dart run build_runner watch --delete-conflicting-outputs`
- **Single test**: `flutter test test/widget_test.dart`
- **Lint**: `dart analyze` (uses `custom_lint` + `riverpod_lint` via `analysis_options.yaml`)
- **Format**: `dart format .`

## Code generation
- `*.g.dart` — generated by `@riverpod` (via `riverpod_generator`) and `@freezed` + `@JsonSerializable` (via `json_serializable`)
- `*.freezed.dart` — generated by `@freezed`
- Always run `build_runner build` after editing files with these annotations; CI or `dart analyze` will catch stale generated code
- Both `.g.dart` and `.freezed.dart` are excluded from analysis (`analysis_options.yaml`)

## Project structure
- **`lib/providers/dio_provider.dart`** — shared Dio instance (`baseUrl: https://bored-api.appbrewery.com/filter`)
- **`lib/pages/`** — 5 self-contained examples, each demonstrating a different `@riverpod` state pattern:
  - `counter/` — plain `int` state with auto-dispose family
  - `enum_activity/` — `@freezed` state class + `ActivityStatus` enum; fetch triggered in `initState`
  - `enum_async_activity/` — same but fetch triggered inside `build()`; no `initial` status
  - `sealed_activity/` — Dart `sealed class` hierarchy (no freezed); fetch triggered in `initState`
  - `sealed_async_activity/` — same but fetch triggered in `build()`
- **`lib/models/activity.dart`** — shared `@freezed` data model, includes `activityTypes` list and `Activity.empty()` factory
- `test/` contains a single smoke test (`widget_test.dart`)

## State management conventions
- All providers use `@riverpod` annotation with code generation
- Every provider prints a dispose message via `ref.onDispose(() => print(...))`
- Fetch providers intentionally fail every other call (`_errorCounter++ % 2 != 1`) to test error UI
- `avoid_print: false` is set in `analysis_options.yaml`—print statements are intentional
- `invalid_annotation_target: ignore` — freezed generates annotations that trigger this; it's expected

## Key dependencies
- `flutter_riverpod` + `riverpod_annotation` + `riverpod_generator` — state management with codegen
- `freezed` + `freezed_annotation` — immutable data classes
- `dio` — HTTP client
- `device_preview_plus` — wraps the app in `DevicePreview` in `main.dart`
- `custom_lint` + `riverpod_lint` — lint rules (run via `dart analyze`)
